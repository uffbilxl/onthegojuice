Critical Findings
Critical Admin Password Stored as Plaintext Cookie Value ✓ Fixed
OWASP: A01 Broken Access Control · A02 Sensitive Data Exposure
Files: src/pages/api/admin/login.js (+ 8 admin routes)

Every admin API route set the HTTP-only cookie value to the literal plaintext admin password, and verified requests by comparing the cookie to process.env.ADMIN_PASSWORD. Any party who could read the cookie — through XSS, network interception, or server log leakage — obtained the admin password directly. The comparison was also a plain === string check, vulnerable to a timing oracle attack.

// BEFORE — cookie contains the actual password serialize('otgj_admin', process.env.ADMIN_PASSWORD, ...) req.cookies?.otgj_admin === process.env.ADMIN_PASSWORD // AFTER — cookie contains an HMAC-signed stateless token // token = "." // verified with crypto.timingSafeEqual()

Fix: Created src/lib/adminAuth.js with HMAC-SHA256 stateless token generation and constant-time verification. Tokens self-expire after 8 hours and cannot be forged without the server secret. All 9 admin routes updated.

Critical No HTTP Security Headers (CSP, HSTS, X-Frame-Options) ✓ Fixed
OWASP: A05 Security Misconfiguration
File: next.config.js

No HTTP security headers were present on any response. The site also broadcast its framework via X-Powered-By: Next.js, aiding attacker reconnaissance. Without these headers: XSS payloads are uncontained, the site is embeddable in iframes (clickjacking), browsers allow MIME-sniffing attacks, and HTTPS is not enforced by the browser.

// ADDED to next.config.js X-Frame-Options: DENY X-Content-Type-Options: nosniff Referrer-Policy: strict-origin-when-cross-origin Permissions-Policy: camera=(), microphone=(), geolocation=(), payment=(self) Strict-Transport-Security: max-age=63072000; includeSubDomains; preload Content-Security-Policy: default-src 'self'; script-src 'self' 'unsafe-inline' https://js.stripe.com https://unpkg.com; ... poweredByHeader: false
Critical Reflected XSS via Unsanitized LocalStorage Cart Data ✓ Fixed
OWASP: A03 Injection (XSS)
File: js/checkout.js · public/js/checkout.js

The checkout page's renderSummary() function built HTML via template literals using raw, unescaped values from localStorage (item.name, item.image). An attacker who can write to localStorage — via a related subdomain, a malicious browser extension, or physical access — can inject arbitrary HTML and execute JavaScript in the victim's browser session, with access to all cookies and session data.

// BEFORE — XSS: item.name = '<img src=x onerror=alert(1)>' <p class="co-summary-item-name">${item.name}</p> <img src="${item.image}" alt="${item.name}" /> // AFTER — all untrusted values HTML-escaped function esc(s) { return String(s).replace(/&/g,'&amp;').replace(/</g,'&lt;') .replace(/>/g,'&gt;').replace(/"/g,'&quot;').replace(/'/g,'&#39;'); } <p class="co-summary-item-name">${esc(item.name)}</p> <img src="${esc(item.image)}" alt="${esc(item.name)}" />
High Findings
High No Rate Limiting on Admin Login Endpoint ✓ Fixed
OWASP: A07 Identification & Authentication Failures
File: src/pages/api/admin/login.js

The /api/admin/login endpoint accepted unlimited password guesses. Combined with the weak password (onthegojuice123), an automated brute-force attack could break into the admin panel within seconds.

Fix: Rate limiter added to adminAuth.js — maximum 10 attempts per IP per 15-minute window. Exceeds limit → HTTP 429. Note: serverless cold-starts reset in-memory state; for hardened production use, replace with a Redis-backed limiter (e.g., Upstash).

High Raw Database Error Messages Exposed to HTTP Clients ✓ Fixed
OWASP: A09 Security Logging & Monitoring Failures
Files: src/pages/api/admin/*.js

Multiple admin API handlers returned the raw Supabase/PostgreSQL error message in the HTTP response body (e.g., "relation 'products' does not exist", "column 'foo' does not exist"). This reveals internal schema structure, table names, and query logic to an attacker who triggers a server error.

// BEFORE return res.status(500).json({ error: error.message }); // AFTER — log server-side, return generic message console.error('[admin/products GET]', error.message); return res.status(500).json({ error: 'Internal server error' });
Medium Findings
Medium IDOR — Unauthenticated Order Cancellation ⚠ Manual Action Required
OWASP: A01 Broken Access Control (IDOR)
File: src/pages/api/cancel-order.js

The cancel-order endpoint only requires a customer's order reference and email address — no session token or authentication. Any attacker who obtains or guesses these values can cancel another customer's active order. Email addresses are often publicly accessible (LinkedIn, business directories), and order references follow a predictable format.

Recommended fix: Issue a single-use signed cancellation token (HMAC or UUID) per order, sent inside the confirmation email. The cancel endpoint should verify this token rather than just matching email + order ref. Alternatively, require the user to be logged in.

Medium Weak Admin Password in Plaintext ⚠ Manual Action Required
OWASP: A07 Identification & Authentication Failures
File: .env.local

The admin password is onthegojuice123 — a predictable pattern that appears in common wordlists and would be cracked in seconds by a dictionary attack. It also serves double duty as the HMAC signing secret for admin tokens after the patch, so its strength directly determines the security of the entire admin session system.

Action required: Rotate to a cryptographically random password of 24+ characters (e.g., openssl rand -base64 32). After rotating, all existing admin sessions automatically invalidate since the HMAC tokens are keyed to the password value.

Low / Informational Findings
Low Framework Version Disclosure via X-Powered-By Header ✓ Fixed
OWASP: A05 Security Misconfiguration

Next.js sets X-Powered-By: Next.js by default. While not directly exploitable, this aids attackers in targeting known framework-specific vulnerabilities. Fixed by setting poweredByHeader: false in next.config.js.

Info All .env.local Secrets Should Be Treated as Potentially Compromised ⚠ Manual Action Required
OWASP: A02 Cryptographic Failures
File: .env.local

The .env.local file contains live Stripe secret keys, the Supabase service role JWT (which has full database bypass permissions), and a Gmail app password. If this file was ever committed to git history, shared via a code review tool, or readable via a misconfigured deployment, all credentials are compromised. Confirm .env.local is in .gitignore and audit git history to ensure it was never committed.

Action required: Rotate all secrets in .env.local from each respective service dashboard (Stripe, Supabase, Google). Treat current keys as compromised until rotated.
